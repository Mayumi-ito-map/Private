## 処理フロー（簡略）

```
excel_local/*.xlsx
    → run_local_build.py（司令塔①）
    → output_match/*_result.xlsx

output_match/*_result.xlsx
    → run_stage_match_quote_variants.py（司令塔②）
    → output_match_results_quote_variants/*_result_result.xlsx

output_match_results_quote_variants/*.xlsx
    → build_geojson_leaflet.py
    → output_leaflet/*.geojson

output_leaflet/*.geojson
    → build_leaflet_index.py
    → output_leaflet/index.json, data_index.js
```

## 主要データの構造

### excel_local（入力 Excel）

- ファイル名: `cn{番号}_{国コード}_{国名}_{3桁コード}.xlsx`
- 必須列: 修正後欧文地名、修正後和文地名、index_ID、num 等
- 緯度経度: local_lat / local_lon、または (lat, long)、(latitude, longitude)、(緯度, 経度) 等

### output_match（司令塔①出力）

- ファイル名: `cn*_*_result.xlsx`
- 主な列: normalized_name、geonames_hits、judge（0/1/2+）、local_lat、local_lon

### output_match_results_*（司令塔②出力）

- ファイル名: `cn*_*_result_result.xlsx`（quote_variants 等）
- 主な列: Stage1_hit、Stage2_hit、Stage3_hit（いずれか1つのみ）、muched、muched_stage

### output_leaflet（地図用）

- **leaflet_map.html**: 単一ページ。data_index.js で国一覧を読み込み、GeoJSON を fetch。
- **data_index.js**: `window.LEAFLET_COUNTRY_INDEX = [{ file, label, cn, region }, ...]`
- **index.json**: `{ "countries": [...] }`（fetch 用フォールバック）
- **cn*_*.geojson**: FeatureCollection。各 Feature は type（Local / GeoNames）、marker-color、original_row 等。

### geonames_master.pkl

- `by_id`: geonameid → レコード（geonameid, name, asciiname, alternatenames, lat, lon, fcl, fcode, ccode 等）
- `by_ccode`: 国コード → [geonameid, ...]
