# run_stage_match.py 設計図報告書

## 1. 概要

`run_stage_match.py` は **司令塔②** として、司令塔①（`run_local_build.py`）で 0マッチ（`judge==0`）となった地名に対し、段階的な正規化（Stage1→Stage2→Stage3）を適用してあいまいマッチングを行うスクリプトである。

正規化エンジンは `normalizers/stage_matcher.py` に分離されており、司令塔②はマッチング制御のみを担当する。

---

## 2. 責務と境界

### 2.1 本スクリプトの責務

| 責務 | 説明 |
|------|------|
| 入力読込 | 司令塔①の出力 Excel（`output_match_alternatenames_cate/*.xlsx`）を読む |
| 対象行の抽出 | `judge==0` の行のみを対象とする |
| 国別 GeoNames 読込 | `cc` 列の2桁国コードから国別に GeoNames データを読み込む |
| Stage マッチング | Stage1→2→3 の正規化を適用し、GeoNames の name/alternatenames と照合 |
| 出力 | マッチング結果を追加列として Excel に書き出す |

### 2.2 本スクリプトの責務外

- **完全一致マッチング**（Phase1/Phase2）→ 司令塔①（`run_local_build.py`）の責務
- **正規化ロジック**（Stage1/2/3 の文字変換）→ `normalizers/stage_matcher.py` の責務
- **GeoNames マスタの構築** → `build_geonames_*.py` の責務

## 5. 処理フロー

### 5.1 メインフロー

```
1. geonames_master.pkl を読み込み（1回のみ）
2. 入力ディレクトリから *.xlsx を順次処理
3. 各ファイルの処理:
   a. Excel 読み込み
   b. judge==0 の行を抽出
   c. cc 列でグループ化
   d. 国コードごとに:
      - GeoNames データ読み込み
      - Stage1/2/3 マップ構築（キャッシュ付き）
      - 対象行を1行ずつマッチング
   e. 結果を Excel に書き出し
4. 全ファイルの統計サマリーを tower2_world_summary.xlsx に出力
```

### 5.2 1行のマッチング処理

```
edited_name 列からマッチング候補を取得（リスト形式）

各候補について:
  Stage1（基本正規化）でマッチ → ヒットなら Stage1 で確定、次の候補へ
  Stage2（実用正規化）でマッチ → ヒットなら Stage2 で確定、次の候補へ
  Stage3（強制正規化）でマッチ → ヒットなら Stage3 で確定、次の候補へ
  全 Stage でヒットなし → still_0

全候補の処理後:
  いずれかの Stage でヒットあり → matched=TRUE, matched_stage=最初にヒットした Stage
  全候補でヒットなし → matched=FALSE
```

### 5.3 Stage マッチの早期終了

1つの候補について、Stage1 でヒットした場合は Stage2/3 は実行しない（上位の Stage で十分な正規化が得られているため）。ただし、**別の候補** は引き続き処理する。

## 7. 正規化エンジン: `normalizers/stage_matcher.py`

### 7.1 3段階の正規化

| Stage | 関数 | 処理内容 | 目的 |
|-------|------|----------|------|
| **Stage1** | `normalize_stage1` | 前後空白除去、全角スペース→半角、連続空白統一 | 見た目の揺れのみ吸収 |
| **Stage2** | `normalize_stage2` | Stage1 + アイン除去 + アクセント除去 + 小文字化 + 記号除去 + スペース除去 | 表記差を吸収して比較可能な形に統一 |
| **Stage3** | `normalize_stage3` | Stage2 + 全角英数字→半角 + 英数字以外除去 | 文字列骨格だけで比較 |

### 7.2 Stage2 のアクセント除去の仕組み

`_remove_accents` 関数は2段階で処理する:

1. **`_SPECIAL_LATIN_MAP`** による `str.translate()` — NFD で分解できない特殊ラテン文字を先に置換
2. **`unicodedata.normalize("NFD")`** で結合記号を分解 → 結合記号（カテゴリ `Mn`）を除去

```python
def _remove_accents(text: str) -> str:
    text = text.translate(_SPECIAL_LATIN_MAP)          # ← 第1段階
    nfd = unicodedata.normalize("NFD", text)           # ← 第2段階
    return "".join(c for c in nfd if unicodedata.category(c) != "Mn")
```

### 7.3 アイン除去

`_remove_ayn` で中東系地名のアイン系記号（`ʻ`, `ʿ`, `'`, `'`, `ʼ`, `ˁ`）を除去。アラビア語・ヘブライ語の地名で表記揺れを吸収する。

## 8. `_SPECIAL_LATIN_MAP` — ダイアクリティカル対応（2026-03-26）

### 8.1 背景

アゼルバイジャン（AZE）の 0マッチ調査で、`allCountries.txt` に完全一致で存在しない5件について、ダイアクリティカル（アクセント記号等）を除去した上で再検索したところ、4件が `alternatenames` から見つかった。残り1件（Dänizkänar）も部分一致（`Danizkanari`、末尾 `i` の差）で発見された。

| ローカル名 | 正規化後 | GeoNames の表記 | マッチ方法 |
|---|---|---|---|
| Siyäzän | siyazan | Siyazan（alt） | ä→a 除去 |
| Xankändi | xankandi | Xankandi（alt） | ä→a 除去 |
| Xocavänd | xocavand | Xocavand（alt） | ä→a 除去 |
| Qazımämmäd | qazimammad | Qazimammad（alt） | ı→i, ä→a 除去 |
| Dänizkänar | danizkanar | Danizkanari（alt） | ä→a 除去 + 末尾差 |

### 8.2 問題点

`_remove_accents` の `unicodedata.normalize("NFD")` は、`ä→a`, `ş→s` 等は NFD で分解可能だが、以下の文字は **独立文字であり NFD では除去できない**。

- `ı`（ドットなし i / U+0131）: トルコ語・アゼルバイジャン語
- `ə`（シュワー / U+0259）: アゼルバイジャン語
- `ð`（エズ）: アイスランド語
- `đ`（クロスド d）: ベトナム語等
- `ł`（ストローク l）: ポーランド語
- `ø`（ストローク o）: ノルウェー語・デンマーク語
- `ß`（エスツェット）: ドイツ語
- その他

### 8.3 対応内容（第1弾: AZE 起因）

`_SPECIAL_LATIN_MAP` マッピングテーブルを追加し、`_remove_accents` の先頭で `str.translate()` を実行するよう変更した。第1弾では AZE 調査で判明した18文字を追加。

| 文字 | 変換先 | 言語例 |
|------|--------|--------|
| `ı` → `i`, `İ` → `I` | ドットなし i | トルコ語・アゼルバイジャン語 |
| `ə` → `a`, `Ə` → `A` | シュワー | アゼルバイジャン語 |
| `ð` → `d`, `Ð` → `D` | エズ | アイスランド語 |
| `đ` → `d`, `Đ` → `D` | クロスド d | ベトナム語等 |
| `ł` → `l`, `Ł` → `L` | ストローク l | ポーランド語 |
| `ø` → `o`, `Ø` → `O` | ストローク o | ノルウェー語・デンマーク語 |
| `ß` → `ss` | エスツェット | ドイツ語 |
| `ħ` → `h`, `Ħ` → `H` | ストローク h | マルタ語 |
| `ơ` → `o`, `ư` → `u`, `Ư` → `U` | ホーン付き | ベトナム語 |

### 8.4 対応内容（第2弾: allCountries.txt 全走査による網羅対応）

第1弾の追加後、`allCountries.txt`（1,340万行）の `name` + `alternatenames` 列を全走査し、NFD で分解できない未対応の特殊ラテン文字を洗い出した。結果、**76種**の未対応文字が見つかり、うち実用上インパクトの大きい **30文字** を追加した。

**北欧（デンマーク語・ノルウェー語・アイスランド語）** — 最もインパクトが大きい

| 文字 | 変換先 | 出現回数 | 備考 |
|------|--------|----------|------|
| `æ` → `ae`, `Æ` → `AE` | 合字 | 62,795 + 666 | 北欧地名で頻出。未対応なら北欧全域で0マッチ増 |
| `þ` → `th`, `Þ` → `TH` | ソーン | 204 + 781 | アイスランド語 |

**フランス語**

| 文字 | 変換先 | 出現回数 |
|------|--------|----------|
| `œ` → `oe`, `Œ` → `OE` | 合字 | 3,208 + 30 |

**西アフリカ諸語（ハウサ語・フラニ語等）**

| 文字 | 変換先 | 出現回数 |
|------|--------|----------|
| `ŋ` → `ng`, `Ŋ` → `NG` | エング | 3,098 + 1 |
| `ɲ` → `n`, `Ɲ` → `N` | パラタル n | 40 + 35 |
| `ɗ` → `d`, `Ɗ` → `D` | フック付き d | 19 + 3 |
| `ɓ` → `b`, `Ɓ` → `B` | フック付き b | — |
| `ƙ` → `k`, `Ƙ` → `K` | フック付き k | 1 + 6 |
| `ƴ` → `y`, `Ƴ` → `Y` | フック付き y | — |

**その他**

| 文字 | 変換先 | 出現回数 | 備考 |
|------|--------|----------|------|
| `ɔ` → `o`, `Ɔ` → `O` | オープン o | 919 | 西アフリカ等 |
| `ɛ` → `e`, `Ɛ` → `E` | オープン e | 616 | 西アフリカ等 |
| `ǝ` → `e` | ターンド e | 339 | カメルーン等 |
| `ɨ` → `i` | ストローク i | 28 | — |

| `ẞ` → `SS` | エスツェット大文字 | 2 | ドイツ語 |

### 8.5 未追加（実害が小さいもの）

出現回数が極めて少ない IPA 記号（`ʒ`, `ɵ`, `ɣ` 等）や全角英字（`Ｃ`, `Ｒ` 等）約46種は、地名マッチングで問題になるケースが限定的なため今回は未追加。今後必要に応じて追加可能。

### 8.6 最終的な `_SPECIAL_LATIN_MAP` の規模

| 区分 | 文字数 |
|------|--------|
| 第1弾（AZE 起因） | 18 |
| 第2弾（全走査追加） | 30 |
| **合計** | **48** |

### 8.7 網羅性の評価

| 対象 | 状態 |
|------|------|
| NFD で分解できる文字（ä, ö, ü, ş, ç, ğ 等） | 元から対応済み |
| NFD で分解できない特殊ラテン文字（48文字） | **第1弾 + 第2弾で対応済み** |
| IPA 記号・極稀な文字（約46種） | 未対応。実害は極めて小さい |

`allCountries.txt` の `name` + `alternatenames` に出現する特殊ラテン文字のうち、**出現回数ベースで 99.9% 以上をカバー**している。
