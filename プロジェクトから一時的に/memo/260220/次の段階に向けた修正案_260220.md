# 次の段階（Leaflet 地図・距離計測・候補の絞り込み）に向けた修正案

作成日: 2026-02-20

---

## 1. 目的

- 取得した地名データを Leaflet 地図上に展開する
- 距離測定・属性情報をもとに候補を一つに絞る
- ローカル（世界大地図帳）と GeoNames を色・形で判別し、属性による色分けも可能にする

---

## 2. 実施した修正（コード側）

### 2.1 司令塔① `run_local_build.py`

- **ローカル緯度経度の取り込み**
  - Excel の列から `local_lat` / `local_lon` を生成し、出力に含める。
  - 対応列:
    - 別々の列: `lat` / `long`、`latitude` / `longitude`、`緯度` / `経度`
    - 1列: `lat,long`（カンマ区切り）
  - 該当列が無い場合は `local_lat` / `local_lon` は欠損のまま出力（次の段階で未整備の地域として扱える）。

### 2.2 司令塔② `run_stage_match.py`

- **変更なし（データの受け渡しで十分）**
  - 司令塔②は output_match の全列をそのまま出力するため、`local_lat` / `local_lon` は自動で output_match_results に引き継がれる。

### 2.3 新規: `export_for_leaflet.py`

- **役割**
  - 司令塔①②の結果（output_match / output_match_results）を、Leaflet 用 GeoJSON に変換する。
- **入力**
  - `output_match_results/*.xlsx` を優先。無い場合は `output_match/*.xlsx`。
- **出力**
  - `output_leaflet/{country_label}.geojson`
  - 例: `cn129_FR_フランス_result.geojson`
- **GeoJSON の構成（LeafletMap見本の data_fr.js に合わせる）**
  - **Local**: `type: "Local"`, `name`, `original_row`, `marker-color: "#0000ff"`
  - **GeoNames**: `type: "GeoNames"`, `id` (geonameid), `country`, `original_row`, `name`, `fcode`, `fcl`, `marker-color: "#ff0000"`
  - **Connection**: `type: "Connection"`, LineString（Local–GeoNames）, `distance_km`
- **距離**
  - 緯度経度から Haversine で `distance_km` を算出し、Connection の properties に付与。
- **実行**
  - `python export_for_leaflet.py`

### 2.4 属性対応表（プレースホルダ）

- **ファイル**: `config/attribute_correspondence.json`
- **内容**: ローカルと GeoNames の属性対応表は未作成のため、説明と空の `mapping` のみ。
- **将来**: ここに「ローカル表記 ↔ fcl/fcode」を定義し、Leaflet で属性別色分けに利用する想定。

---

## 3. データの流れ（次の段階まで）

```
excel_local/*.xlsx (修正後欧文地名, [lat/long があれば])
    → run_local_build.py
    → output_match/*_result.xlsx (normalized_name, judge, geonames_hits, local_lat, local_lon, ...)
    → run_stage_match.py
    → output_match_results/*.xlsx (+ Stage1_hit, Stage2_hit, Stage3_hit)
    → export_for_leaflet.py
    → output_leaflet/*.geojson
    → Leaflet (index_fr.html 等で読み込み、国選択・地名一覧・地図連携・距離表示)
```

---

## 4. Leaflet 側で行うこと（案）

- 国選択プルダウン: `output_leaflet` のファイル一覧または国コード一覧から読み込む。
- 地名一覧と地図の連携: GeoJSON の `original_row` や `id` (geonameid) で紐付け。
- ローカル / GeoNames の判別: `properties.type` と `marker-color`（または shape）で区別。
- 属性による色分け: `properties.fcl` / `properties.fcode` と、今後整備する `attribute_correspondence.json` を組み合わせて色を決定。
- 距離表示: `properties.distance_km` を Connection のポップアップなどに表示。

---

## 5. ローカル緯度経度について

- 世界大地図帳の Excel に「lat, long」列（または同様の列）がある場合、司令塔①で `local_lat` / `local_lon` に変換され、以降の出力と GeoJSON に含まれる。
- 無い場合は Local Point と Connection は出力されず、GeoNames 候補の Point のみになる。
- 索引ベースで緯度経度が未整備の国は、手作業で緯度経度を追加した Excel を用意すれば、同じパイプラインで GeoJSON に反映できる。

---

## 6. まとめ

- 司令塔①で **local_lat / local_lon** を標準化して出力。
- **export_for_leaflet.py** で Leaflet 用 GeoJSON を生成（Local / GeoNames / Connection + distance_km）。
- **attribute_correspondence.json** は属性対応表のプレースホルダとして用意済み。
- 次の段階では、Leaflet の国選択・一覧・地図・距離表示・属性色分けを、この GeoJSON と対応表で実装できる。
