---
1. 2文字でも「対象 0」ではなかった
judge==0 の 238 行（7%） に、' または ' が含まれている。
なので 2文字版でも、記号ゆれの展開は 238 行分はかかっており、候補数は Base より増えていた可能性がある。
「Base と全く同じ」だったのは、行数（rows_with_hit / rows_still_0） だけを見た比較で、候補数（total_candidates） は増えていたかもしれない、という意味になる。
2. 地域の偏り
例は アラブ首長国連邦・イエメン・イスラエル など西アジア。
地名中の ' はアラビア語の ʻayn などの転写でよく使われる。
記号ゆれの効果が出やすいのは、こうした地域の judge==0 の行、という理解でよい。
3. 2文字で行数が増えなかった理由の推測
238 行では候補は増えていたが、増えた候補が GeoNames の表記と一致しなかった可能性が高い。
例: ローカルは '、GeoNames は ʿ (U+02BF) で登録 → ' と ' の 2 文字だけでは「ʿ にそろえた候補」を作らないのでマッチしない。
7 文字なら ʿ も対象なので、「ʿ にそろえた候補」を試せ、西アジアでヒットが増える説明と合う。
4. openpyxl の警告
Cannot parse header or footer so it will be ignored は、Excel のヘッダー／フッターを解釈できなかったというだけの警告で、データの読み取りには影響しません。無視して問題ない。

---
結論:   
judge==0 の 7% には確かに ' または ' が含まれており、2文字版でもここは展開されていた。それでも行数が増えなかったのは、GeoNames 側が ʿ 等で登録しており、2文字ではその候補を試せていなかったためと考えられ、7文字に戻している現状の設定はこの診断結果と整合する、という整理にした。
