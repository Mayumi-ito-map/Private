# 一般語・接頭辞/接尾辞の同義語展開 — 解説と仕様

## 1. 目的

司令塔①で `judge==0`（完全一致しなかった）行は、司令塔②で Stage1→2→3 の段階的正規化マッチングにかける。  
それでもマッチしない要因の一つが **「表記の違いで同じ地名」** である。

- 例: ローカル「Mt. Everest」 ↔ GeoNames「Mount Everest」
- 例: ローカル「St. John's」 ↔ GeoNames「Saint John's」
- 例: ローカル「Jebel An-Nabi」 ↔ GeoNames「Jabal an Nabi」

これらは **正規化（空白・記号除去など）だけでは揃わない** ため、**候補そのものを増やす** 必要がある。

## 2. アプローチの概要

**「一般語・接頭辞/接尾辞の同義語展開」** とは、  
地名に含まれる **山・聖・湖・川・岬・海峡** などの **共通語** について、

- 略形（Mt., St., J., L., R., C., Pt., B., G., I., Is., Str. など）
- フルスペル（Mount, Saint, Jebel/Jabal, Lake, River, Cape, Point, Bay, Gulf, Island, Strait など）
- 言語・表記ゆれ（Jebel / Jabal / Djebel / J.）

を **同義** とみなし、**1つの候補から複数の表記候補を生成する** こと。

- 入力: 候補 1 つ（例: `"Mt. Everest"`）
- 出力: 候補リスト（例: `["Mt. Everest", "Mount Everest", "Mt Everest"]`）

このリストを **Stage マッチングの入力** に使う。いずれか 1 つでも GeoNames の name/asciiname/alternatenames と一致すれば、その行はヒットとみなす。

## 3. 実装の置き場所（司令塔②に追加する関数）

- **司令塔①（run_local_build.py）**: 変更しない。略語は既存の「略語変換表.xlsx」で 1→N 候補を出している。
- **司令塔②（run_stage_match.py）**: ここで **追加の関数** として同義語展開を使う。

流れ:

1. `output_match` の `judge==0` の行を読む。
2. 列 `normalized_name` から候補リストを取得（`parse_normalized_name`）。
3. **そのリストの各要素に対して同義語展開** し、候補を増やす（`expand_candidates_synonyms`）。
4. 増えた候補リストで Stage1 → Stage2 → Stage3 のマッチングを行う。

つまり「**司令塔②専用の、候補を増やす前処理**」として、**normalizers にモジュールを追加**し、司令塔②から呼ぶ形にしている。

- **追加モジュール**: `normalizers/expand_synonyms.py`
  - `expand_place_name_synonyms(text)` … 1 候補 → 同義語展開したリスト
  - `expand_candidates_synonyms(candidates)` … 複数候補をまとめて展開し重複除去
- **司令塔②**: `parse_normalized_name` の直後に `expand_candidates_synonyms` を呼ぶ。

## 4. どの語を「同義」とするか（現在の定義）

`normalizers/expand_synonyms.py` の `SYNONYM_PREFIXES` で、**接頭辞** の対応を定義している。

| カテゴリ | 接頭辞例 | 展開例 |
|----------|----------|--------|
| 山 | Mt., Mount, Mt | Mt. X → Mount X, Mt X |
| 聖・サン | St., Saint, St | St. X → Saint X, St X |
| 山（アラビア系） | Jebel, Jabal, J., Djebel | Jebel X ↔ Jabal X ↔ J. X ↔ Djebel X |
| 湖 | L., Lake, L | L. X → Lake X, L X |
| 河川 | R., River, R | R. X → River X, R X |
| 岬 | C., Cape, C | C. X → Cape X, C X |
| 岬/ポイント | Pt., Point, Pt | Pt. X → Point X, Pt X |
| 海峡 | Str., Strait, Str | Str. X → Strait X, Str X |
| 湾 | B., Bay, B | B. X → Bay X, B X |
| 湾（大きな） | G., Gulf, G | G. X → Gulf X, G X |
| 島 | I., Is., Island, I | I. X → Island X, Is. X, I X |

- **接頭辞** のみ対象（語頭、または空白後の語頭で一致する部分を置換）。
- **元の表記は必ず残す**（削除しない）。略語展開と同様「元 + 展開形」を候補に含める。
- 必要に応じて、ここに **接尾辞**（例: -ville, -burg）や **言語別の綴り**（例: -berg / -burg）を追加できる。

## 5. 行単位指標との関係

- **候補単位**: 従来どおり `stage1` / `stage2` / `stage3` / `still_0`（候補数と %）。
- **行単位**:  
  - `rows_with_hit` … judge==0 の行のうち、Stage1/2/3 の **どれかで 1 件でもヒットした行数**  
  - `rows_still_0` … 行単位でまだ 1 件もヒットしていない行数（`total_0_target - rows_with_hit`）。

最終的に「まだ手付かず」なのは **行** なので、`rows_still_0` が still_0 の行数として必須。  
同義語展開は、その `rows_still_0` を減らす（行あたりのヒット率を上げる）ための施策。

## 6. 運用上の注意

- **略語変換表.xlsx** と役割が違う: 略語変換表は「世界大地図帳固有の略語」、同義語展開は「汎用的な地名接頭辞の表記ゆれ」。
- 同義語の組み合わせは **コードで管理**（`SYNONYM_PREFIXES`）。語を増やしたい場合はここを編集する。
- 言語・地域ごとに語を増やす場合（例: ドイツ語 -berg/-burg、スペイン語 Isla/Is. など）も、同じリストに追加する形で拡張できる。
