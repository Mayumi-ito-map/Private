# 記号揺れ（2文字版・7文字版）と同義語展開の概要

司令塔②で judge==0 の候補を「増やしてから」Stage1→2→3 でマッチングする3つの方式の概要です。

---

## 1. 記号揺れ 7文字版

### 概要

地名に含まれる**アポストロフィ・引用符・アイン等**を、互いに置換した候補を追加する。symbol_summary では ' と ' のみだったが、司令塔②の入力には ʻ ʿ 等が含まれる場合があり、7文字にしておくと候補が増えて効果が出る。

### 対象文字（QUOTE_LIKE_CHARS）

| 文字 | Unicode | 説明 |
|------|---------|------|
| ' | U+0027 | ASCII apostrophe |
| ' | U+2019 | RIGHT SINGLE QUOTATION MARK |
| ` | U+0060 | grave accent |
| ʻ | U+02BB | MODIFIER LETTER TURNED COMMA（ハワイ語 ʻokina、アラビア語 hamza 等） |
| ʿ | U+02BF | MODIFIER LETTER LEFT HALF RING（アラビア語 ʿayn 等） |
| ´ | U+00B4 | acute accent |
| ʼ | U+02BC | MODIFIER LETTER APOSTROPHE |

### 処理例

- 入力: `"d'Or"`（ʻ を含む）
- 出力候補: `"d'Or"`, `"d'Or"`, `"d'Or"` 等（各記号に統一した候補を追加）

### スクリプト・モジュール

- **run_stage_match_quote_variants.py**（import: `expand_quote_variants`）
- **normalizers/expand_quote_variants.py**
- **出力先**: `output_match_results_quote_variants`

---

## 2. 記号揺れ 2文字版

### 概要

symbol_summary.xlsx の結果（' と ' のみ出現）に合わせた版。比較用に別ファイルで保持。7文字版より候補が少ないため、GeoNames が ʿ 等で登録している場合はマッチしにくい。

### 対象文字（QUOTE_LIKE_CHARS）

| 文字 | Unicode |
|------|---------|
| ' | U+0027 |
| ' | U+2019 |

### 処理例

- 入力: `"d'Or"`
- 出力候補: `"d'Or"`, `"d'Or"`（最大 3 種類）

### スクリプト・モジュール

- **run_stage_match_quote_variants.py** で import を `expand_quote_variants_2chars` に差し替え、OUTPUT_DIR を変更して実行
- **normalizers/expand_quote_variants_2chars.py**
- **出力先**: `output_match_results_quote_variants_2chars`

### 7文字版との違い

- 2文字版: ' と ' のみ。judge==0 の約 7% に ' または ' が含まれるが、行数ベースの結果は Base と同程度（GeoNames が ʿ 等で登録しているため）。
- 7文字版: ʻ ʿ 等も含むため、西アジア等でヒットが増える。

## 3. 同義語展開

### 概要

地名の**一般語・接頭辞の同義語**を候補として追加する。Mt./Mount、St./Saint、Jebel/Jabal、Lake/L. 等の略語・フルスペルを相互に展開する。

### 対象（SYNONYM_PREFIXES の例）

| カテゴリ | 例 |
|----------|-----|
| 山 | Mt. ↔ Mount ↔ Mt |
| 聖人・サン | St. ↔ Saint ↔ St |
| アラビア系「山」 | Jebel ↔ Jabal ↔ J. ↔ Djebel |
| 湖・河川・湾等 | L. ↔ Lake、R. ↔ River、C. ↔ Cape、Pt. ↔ Point、Str. ↔ Strait、B. ↔ Bay、G. ↔ Gulf |
| 島 | I. ↔ Island ↔ Is. |

### 処理例

- 入力: `"Mt. Everest"`
- 出力候補: `["Mt. Everest", "Mount Everest", "Mt Everest"]`
- 入力: `"St. John's"`
- 出力候補: `["St. John's", "Saint John's", "St John's"]`

### スクリプト・モジュール

- **run_stage_match_同義語展開.py**（import: `expand_candidates_synonyms`）
- **normalizers/expand_synonyms.py**
- **出力先**: `output_match_results_同義語のみ`（または output_match_results）

---

## 3つの比較

| 方式 | 候補の増やし方 | 主な効果が出る場面 |
|------|----------------|---------------------|
| **記号揺れ 7文字版** | アポストロフィ・引用符・アイン等を互いに置換 | 西アジア等、ʻ ʿ を含む地名 |
| **記号揺れ 2文字版** | ' と ' のみ置換 | 比較用。効果は 7文字版より小さい |
| **同義語展開** | Mt./Mount、St./Saint、Jebel/Jabal 等の略語・フルスペル | 山・湖・聖人地名等 |

### ベースラインとの関係

- **run_stage_match_baseline.py**: 記号ゆれも同義語も行わない。Stage1→2→3 のみ。
- 記号ゆれ版・同義語版は、ベースラインの「前」に候補を追加してから、同じ Stage1→2→3 でマッチングする。
