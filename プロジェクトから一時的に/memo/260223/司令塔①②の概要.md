# 司令塔①・司令塔②の概要

地名マッチングを2段階で行う処理の概要です。

---

## 司令塔①（run_local_build.py）

### 役割

ローカル Excel の地名候補を生成し、GeoNames と**完全一致**でマッチングする。

### 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `excel_local/*.xlsx`（国別 Excel、cn002～cn516） |
| **出力** | `output_match/*_result.xlsx`、`output_match/world_stats.xlsx` |

### 処理の流れ

1. **対象列**: 「修正後欧文地名」を読み込む。
2. **候補生成**: `normalize_comma_abb` でカンマ・略語を正規化し、1→N の候補を生成。
3. **国コード**: ファイル名から国コードを取得。`config/overseas_territories.json` で海外領土を考慮。
4. **マッチング**: `geonames_master.pkl`（by_id / by_ccode）から国別に地名を取得し、**完全一致**で判定。
5. **結果**: geonameid でユニーク化し、`judge` を付与。
   - **judge = "0"**: ヒット 0 件
   - **judge = "1"**: ヒット 1 件
   - **judge = "2+"**: ヒット 2 件以上
6. **緯度経度**: pkl の lat/lon を保持。local_lat / local_lon を Excel から取得（地図・距離計測用）。

### 主な出力列

- `normalized_name`（候補リスト）
- `geonames_hits`（マッチした GeoNames レコードのリスト）
- `judge`（0 / 1 / 2+）
- `local_lat`, `local_lon`（ローカル座標）

---

## 司令塔②（run_stage_match_*.py）

### 役割

司令塔①で **judge == "0"**（ヒット 0 件）の行だけを対象に、**段階的正規化（Stage1→2→3）** でマッチングする。必要に応じて、記号ゆれや同義語の候補を追加してからマッチングする。

### 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `output_match/*_result.xlsx`（司令塔①の結果） |
| **出力** | `output_match_results_*/*_result.xlsx`、`world_summary.xlsx` |

### 処理の流れ

1. **対象**: `judge == "0"` の行のみ。judge != "0" は司令塔①でマッチ済みのためスキップ。
2. **候補の取得**: `normalized_name` から候補リストを取得。
3. **候補の追加**（スクリプトによる）:
   - 記号ゆれ版: アポストロフィ・引用符等の記号ゆれ候補を追加
   - 同義語版: Mt./Mount、St./Saint 等の同義語候補を追加
   - ベースライン: 追加なし
4. **段階的マッチング**: 各候補について Stage1 → Stage2 → Stage3 の順で試し、最初にヒットした段階で終了。
5. **結果の記録**: 行単位で「最初にヒットした段階」だけを記録。
   - `Stage1_hit` / `Stage2_hit` / `Stage3_hit` のいずれか 1 つのみにデータを入れる。
   - `muched`（当たったか）、`muched_stage`（stage1 / stage2 / stage3）を付与。

### Stage1 / Stage2 / Stage3 の違い

| Stage | 正規化の強さ | 主な処理 |
|-------|--------------|----------|
| **Stage1** | 弱い | 前後空白除去、全角→半角スペース、連続空白を1つに。形をほぼ保つ。 |
| **Stage2** | 中 | Stage1 + アイン除去、アクセント除去、小文字化、括弧・ダッシュ除去、スペース除去。 |
| **Stage3** | 強い | Stage2 + 全角→半角（NFKC）、英数字・アンダースコア以外除去。骨格のみで比較。 |

### 主な出力列

- `Stage1_hit` / `Stage2_hit` / `Stage3_hit`（ヒットした段階の GeoNames レコードリスト、1行に1段階のみ）
- `muched`（True/False）
- `muched_stage`（"stage1" / "stage2" / "stage3" / 空欄）

---

## 2つの司令塔の関係

```
excel_local/*.xlsx
    ↓ 司令塔①（完全一致）
output_match/*_result.xlsx  [judge: 0 / 1 / 2+]
    ↓ 司令塔②（judge==0 のみ、Stage1→2→3）
output_match_results_*/*_result.xlsx  [Stage1/2/3_hit, muched, muched_stage]
```

- **司令塔①**: 完全一致で一括判定。judge でヒット件数を付与。
- **司令塔②**: judge==0 の行だけを、段階的正規化で再マッチング。記号ゆれ・同義語の効果を確認するために、複数バリエーションのスクリプトがある。
